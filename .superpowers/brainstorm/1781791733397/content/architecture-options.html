C. 完整数据平台

并发与运维能力最强，但对本地单用户、单 GPU 场景明显过重。

推荐 B 的原因

  • 覆盖实时进度、历史回放和双版本对比
  • 保留 SQLite/FAISS/Ollama 现有投资
  • 未来可替换队列或数据库，不阻塞首版

明确代价

  • 必须把测试脚本重构成可调用流水线
  • 需要新增 Run 数据模型和事件协议
  • 需要维护 React 前端与 Docker Compose
